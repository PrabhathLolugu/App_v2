2. Create MyItihasRepository class extending OfflineFirstWithSupabaseRepository:
   - Private constructor with named super parameters: supabaseProvider, sqliteProvider, migrations, offlineRequestQueue
   - Static late _singleton field
   - Static configure() method that:
     * Uses clientQueue() helper with databaseFactory and ignorePaths {'/auth/v1', '/storage/v1', '/functions/v1'}

     * Creates SqliteProvider with 'myitihas.sqlite' filename and databaseFactory
     * Sets modelDictionary to empty {} for both providers (will be populated when models are created)
     * Sets migrations to empty {} (will be generated later)
   - Static instance getter returning _singleton

3. DO NOT import brick.g.dart yet - it doesn't exist until models are created and build_runner runs

Follow EXACTLY the pattern in RESEARCH.md "Repository Skeleton (Before Models Exist)" code example. This skeleton allows compilation without generated files.

CRITICAL: Use `import 'package:sqflite/sqflite.dart' show databaseFactory;` - the "show databaseFactory" is required to avoid namespace conflicts.
