**Rationale for memCacheWidth/memCacheHeight:**
RESEARCH.md Pattern 2 recommends these parameters to reduce memory footprint. Without them, full-resolution images (e.g., 4000x3000) are kept in memory, causing crashes on low-end devices. Setting memCacheWidth=800 resizes to 800px width before storing in ImageCache, reducing memory usage by ~80-90%.

**Parameter values by widget type:**
- Feed/story thumbnails: memCacheWidth=800, memCacheHeight=800
- Avatar images: memCacheWidth=200, memCacheHeight=200
- Full-screen images: memCacheWidth=1200, memCacheHeight=1200

**2. Add prefetch call in event handler after loading posts:**

Find the event handler that loads posts (likely `_onLoadFeed` or similar). After emitting success state with loaded posts, add:

```dart
on<_LoadFeed>((event, emit) async {
  emit(const FeedState.loading());

  final result = await _postRepository.getPosts(/* ... */);

  result.fold(
    (failure) => emit(FeedState.error(failure.message)),
    (posts) {
      emit(FeedState.loaded(posts));

      // Prefetch images for offline availability
      final imageUrls = posts
          .whereType<ImagePost>()
          .map((post) => post.imageUrl)
          .where((url) => url.isNotEmpty)
          .toList();

      if (imageUrls.isNotEmpty && event.context != null) {
        _prefetchService.prefetchImages(
          event.context!,
          imageUrls,
          maxPrefetch: 15,  // Budget: prefetch first 15 images
        ).catchError((e) {
          talker.error('Feed prefetch failed', e);
        });
      }
    },
  );
});
```

**CRITICAL:** PrefetchService requires BuildContext. If FeedEvent doesn't include context, add it:
```dart
@freezed
sealed class FeedEvent with _$FeedEvent {
  const factory FeedEvent.loadFeed({BuildContext? context}) = _LoadFeed;
  // ... other events
}
```

**Why fire-and-forget:** Prefetch runs asynchronously with catchError to prevent failures from affecting feed display. User sees posts immediately, prefetch happens in background.

**Budget (maxPrefetch: 15):** Based on RESEARCH.md recommendation to limit prefetch to 10-20 items. Feed shows ~10 posts per screen, so 15 ensures next screen worth is cached.
