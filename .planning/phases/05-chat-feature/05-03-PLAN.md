Constructor injection (add):
- final MyItihasRepository brickRepository;
- final InternetConnection internetConnection;

State management:
- Add field: StreamSubscription<List<MessageModel>>? _messageSubscription;
- Add event: RealtimeMessageReceivedEvent(Message message) in chat_detail_event.dart
- Update ChatDetailState to include: String? error, bool isOfflineError (for snackbar detection)

Implementation:

1. _onLoadMessages event handler:
   - After successful message load, call _startRealtimeSubscription(conversationId)

2. _startRealtimeSubscription(String conversationId):
   - Subscribe: _messageSubscription = brickRepository.subscribeToRealtime<MessageModel>(query: Query.where('conversationId', conversationId)).listen((messages) { ... })
   - In listener: if messages.isNotEmpty, add(ChatDetailEvent.realtimeMessageReceived(messages.first.toDomain()))

3. _onRealtimeMessage event handler:
   - Check state is ChatDetailLoaded
   - Insert new message at top: updatedMessages = [event.message, ...currentState.messages]
   - Emit currentState.copyWith(messages: updatedMessages)

4. _onSendMessage event handler:
   - Check connectivity: final isOnline = await internetConnection.hasInternetAccess;
   - If offline: emit currentState.copyWith(error: 'No internet connection. Try again later.', isOfflineError: true); return;
   - Otherwise: continue with existing send logic

5. Override close():
   - await _messageSubscription?.cancel();
   - return super.close();

Keep existing event handlers, just enhance with connectivity checks and realtime subscription.
