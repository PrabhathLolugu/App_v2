Add Brick realtime subscriptions for live feed updates:

1. In FeedBloc constructor (after connectivity subscription):
   - Call private method _startRealtimeSubscriptions()

2. Create _startRealtimeSubscriptions() method:
   ```dart
   void _startRealtimeSubscriptions() {
     // Subscribe to image posts
     _imagePostSubscription = _repository
       .subscribeToRealtime<ImagePostModel>()
       .listen((posts) {
         if (posts.isNotEmpty) {
           add(FeedEvent.realtimePostReceived(
             FeedItem.imagePost(posts.first.toDomain())
           ));
         }
       });

     // Subscribe to text posts
     _textPostSubscription = _repository
       .subscribeToRealtime<TextPostModel>()
       .listen((posts) {
         if (posts.isNotEmpty) {
           add(FeedEvent.realtimePostReceived(
             FeedItem.textPost(posts.first.toDomain())
           ));
         }
       });

     // Subscribe to video posts
     _videoPostSubscription = _repository
       .subscribeToRealtime<VideoPostModel>()
       .listen((posts) {
         if (posts.isNotEmpty) {
           add(FeedEvent.realtimePostReceived(
             FeedItem.videoPost(posts.first.toDomain())
           ));
         }
       });

     talker.info('[FeedBloc] Realtime subscriptions started');
   }
   ```

3. Import requirements:
   - ImagePostModel, TextPostModel, VideoPostModel from data/models
   - FeedItem from domain/entities

WHY subscribeToRealtime: Brick handles Supabase realtime and auto-upserts to cache
WHY check posts.isNotEmpty: Stream can emit empty list on connection changes
WHY posts.first: Most recent post based on default ordering
WHY _repository field name: Consistent naming across all tasks
